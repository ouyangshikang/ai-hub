WAIC 2026 闭幕：WAICO 成立，351 款产品全球首发，意向采购约 203.6 亿元
2026 世界人工智能大会 7 月 20 日在上海闭幕，同期成立世界人工智能合作组织（WAICO），29 国参与。

#12
OpenAI 确认 GPT-5.6 可能误删文件，称之为"诚实的错误"
OpenAI 承认 GPT-5.6 在执行文件操作任务时可能意外删除文件，再次凸显 AI Agent 操作文件系统的安全防护需求。

OpenAI 于 7 月 19 日确认 GPT-5.6 在文件操作任务中存在误删文件问题，官方将其定性为 "honest mistake"（诚实的错误）。事件在开发者社区引发广泛讨论：当 AI 编码代理拥有文件系统写入权限时，如何设计有效防护机制防止不可逆操作。

#14
MCP 规范 2026-07-28 版本进入 RC：协议核心变为无状态
MCP 协议即将发布重大版本更新：核心架构变为无状态设计，新增 MCP Apps 和 Tasks 扩展，废弃 Roots/Sampling/Logging。

MCP 官方博客已发布 2026-07-28 版本的 Release Candidate（正式版定于 7 月 28 日发布）。此次是 MCP 自发布以来最大的架构变更：协议核心变为无状态（stateless），取消初始化和会话跟踪机制，请求可由任意实例处理；客户端/版本信息移入逐请求元数据。

新特性包括：扩展机制正式化，"MCP Apps" 允许在沙箱 iframe 中渲染 HTML UI；重新设计的 "Tasks" 扩展支持长时操作。授权机制强化了 issuer 验证和凭证绑定。工具 Schema 采用 JSON Schema 2020-12。Roots、Sampling、Logging 被标记为废弃。无状态设计大幅简化了 MCP 服务器的水平扩展和负载均衡部署。

#15
Claude Code 悄然切换至 Bun 的 Rust 重写版本
Anthropic 将 Claude Code 运行时切换为 Bun 的 Rust 移植版，Linux 启动速度提升 10%，标志 Rust 在 AI 工具链中的渗透。

据 Simon Willison 7 月 19 日博文披露，Bun 作者 Jarred Sumner 确认 Claude Code v2.1.181 及之后版本已使用 Bun 的 Rust 移植版作为运行时。Willison 检查自己的安装后确认发现了 "Bun v1.4.0 (macOS arm64)" 及大量 Rust 源文件路径。

性能方面 Linux 上启动速度提升约 10%，其他平台变化不大，"几乎没人注意到"。这一无感迁移验证了 Bun 从 Zig 到 Rust 的大规模重写（96 万行代码，由 Claude 辅助完成）已具备生产级稳定性，也反映了 AI 编码工具向 Rust 生态靠拢的趋势。

#16
BigMoeOnEdge：中端安卓手机 CPU 上运行 120B 参数 MoE 模型
开源项目 BigMoeOnEdge 实现仅用 CPU 在中端安卓手机上推理 120B 参数 MoE 模型，展示端侧大模型部署新可能。

开发者 Helldez 在 GitHub 发布 BigMoeOnEdge 项目，使用 llama.cpp 在中端安卓手机纯 CPU 环境下运行 120B 参数 MoE 模型。该项目利用 MoE 架构的稀疏激活特性（实际激活参数远小于总参数量），结合激进量化策略，将超大模型压缩到移动设备可承受范围。

虽然推理速度较慢，但对离线场景、隐私敏感场景和边缘计算应用具有重要参考价值。结合 NVIDIA 同日发布 Cosmos 3 Edge（4B 世界模型面向边缘设备），端侧 AI 部署正从"小模型"向"大模型"快速演进。

#17
Bothread：多 AI 编码代理共享代码仓库的协作框架
开源项目 Bothread 让多个 AI 编码代理在同一代码仓库中协作对话并协调修改，解决多 Agent 并行编码的合并冲突问题。

Bothread 允许多个 AI 编码代理在同一代码仓库中进行对话式协作，各自承担不同任务的同时协调代码修改，避免产生合并冲突。这解决了当前多 Agent 编码工作流中的核心痛点：多个 Agent 并行修改同一代码库时如何保持一致性。

多 Agent 协作编码正成为 2026 年重要趋势（微软 Agent Framework、Meta Muse Spark 1.1 的并行子代理委派等都指向这一方向）。Bothread 提供了轻量级开源方案，适合希望实验多 Agent 软件工程工作流的开发者。
