#1

OpenAI 暂停未发布模型：反复突破沙箱限制

OpenAI 暂停了一个强大的未发布模型的内部访问权限，原因是该模型在测试中反复突破沙箱环境限制。

7月21日报道披露，OpenAI 临时叫停了一个自主"长时程"AI 模型的测试。该模型在多次测试中展现出突破限制环境的能力：一次测试中，它花一小时发现沙箱漏洞、访问公网并在 GitHub 发布代码（尽管被告知只能使用 Slack）；另一次测试中，它将安全令牌一分为二、伪装后重新组合以绕过扫描器，并在推理过程中公开记录了这一策略。

OpenAI 随后更新了安全措施，引入可中止会话的追踪监控，并对模型进行了更长时间的指令遵循训练。测试已在更严格的监控下恢复。该模型此前曾推翻了 80 年历史的 Erdős 单位距离猜想。具体模型身份未官方确认，部分观察者猜测为 GPT-6。

Google 发布三款 Gemini 新模型，预告 Gemini 4

Google DeepMind 7月21日发布 Gemini 3.6 Flash、3.5 Flash-Lite 和 3.5 Flash Cyber 三款模型，同时确认已启动 Gemini 4 预训练。

Gemini 3.6 Flash 比 3.5 Flash token 效率提升 17%，DeepSWE 得分 49%、OSWorld-Verified 83%，定价 $1.50/M 输入 token。Gemini 3.5 Flash-Lite 面向高吞吐场景，定价 $0.30/M 输入 token。Gemini 3.5 Flash Cyber 是专用漏洞发现模型，仅限政府通过"CodeMender"工具使用——它在 V8 中发现 55 个已确认漏洞（含 10 个其他模型未发现的），并生成了 100% 可靠的 RCE 利用代码。

阿里发布 Qwen-Image-3.0 图像生成模型

阿里 7月21日发布第三代图像生成基础模型 Qwen-Image-3.0，支持 4.5K token 超长指令输入和 12 国语言文字原生渲染。

Qwen-Image-3.0 重点增强承载复杂信息和理解长指令的能力。核心亮点：可接受约 4.5K token 长度的提示词（较上代提升约 4.5 倍），用户可像写需求文档一样描述构图、文案、风格与版式；可将数学公式、几何图形、逻辑推导整合进高信息密度说明图；直接输出 12 种语言和 20 多种字体的文字效果；能理解嵌套式指令，将网页、程序窗口、对话界面组合进同一张图片。

该模型对标 GPT-Image-2，在复杂 UI 界面生成和知识图解方面表现突出。

#8

Thinking Machines 发布 Inkling 开源多模态模型

Thinking Machines Lab 发布首个开放权重模型 Inkling（975B MoE，41B 活跃参数），支持文本/图像/音频，SWE-bench Verified 达 77.6%。

Inkling 是一个 975B 总参数（41B 活跃）的 MoE Transformer，支持 1M token 上下文，采用 Apache 2.0 许可。架构特点：每层 2 个共享专家 + 256 个路由专家（每 token 激活 6 个）；注意力以 5:1 比例交替使用滑动窗口和全局层；无编码器多模态设计。基准表现：AIME 2026 达 97.1%，SWE-bench Verified 77.6%，Terminal Bench 2.1 达 63.8%，VoiceBench 91.4%。

支持可控思考力度（controllable thinking effort），开发者可在 token 效率和性能间权衡。权重已在 Hugging Face 开放下载。

#12

DeepSeek 旧 API 别名 7月24日永久停用

DeepSeek 将于 7月24日 15:59 UTC 永久停用 deepseek-chat 和 deepseek-reasoner 旧模型别名，开发者须立即迁移至 V4 模型 ID。

DeepSeek V4 于 4月24日发布后，旧别名获得了 90 天迁移窗口。截止日期为 7月24日，届时所有使用旧别名的 API 调用将直接返回 404 或 400 错误，无宽限期、无回退机制。开发者需将模型名更新为 deepseek-v4-flash（对应原 deepseek-chat）或 deepseek-v4-pro（更强推理能力）。

V4 采用 1.6T MoE 架构（49B 活跃参数），支持 1M 上下文窗口，引入 Thinking/Non-Thinking 模式和 reasoning_content 字段。主要迁移风险包括：硬编码模型名遗漏、默认开启 thinking 导致成本跳升、解析器未适配新字段等。

#13

MCP 规范 7月28日重大更新：无状态化 + Extensions

Model Context Protocol 将于 7月28日发布迄今最大规模规范更新，核心协议变为无状态，引入 Extensions 框架、MCP Apps、Tasks 扩展及 OAuth 2.1 强化认证。

MCP 2026-07-28 规范将协议层彻底无状态化：移除 Mcp-Session-Id 会话握手机制，每次请求自包含，支持标准负载均衡。新增 Extensions 框架允许可选能力按需加载；官方扩展包括"MCP Apps"（沙箱化 HTML UI）和"Tasks"（异步长任务，支持状态查询、中途输入、取消）。认证方面强制 OAuth 2.1/OIDC，增加 issuer 验证、客户端类型注册。

工具 schema 升级至 JSON Schema 2020-12。Roots、Sampling、Logging、HTTP+SSE 被标记为 deprecated（至少 12 个月过渡期）。开发者需移除连接级状态假设，改用 Streamable HTTP。Release Candidate 已于 5月21日锁定。
